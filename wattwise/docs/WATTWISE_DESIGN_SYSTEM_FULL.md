# WattWise — Design System (TEHSO Production Standard)

## 0. Foundation

WattWise follows the TEHSO design philosophy:

"The interface should disappear. The learning should remain."

This system is strict. Deviation creates inconsistency and reduces product quality.

---

## 1. Core Design Principles

### 1.1 Clarity Over Decoration
- No visual noise
- No unnecessary UI elements
- Every element must serve a purpose

### 1.2 One Accent Rule
- Only ONE primary accent color is used across the system

### 1.3 White Space First
- Layout begins with spacing, not components

### 1.4 Typography Leads
- Hierarchy is driven by text, not color

---

## 2. Color System

### Primary Color
- TEHSO Blue: #2E53FF

### Neutrals
- Background: #FFFFFF
- Surface: #F5F5F5
- Divider: #EBEBEB
- Text Primary: #1A1A1A
- Text Secondary: #6B6B6B
- Text Muted: #A0A0A0

---

## 3. Color Usage Rules

### Allowed Blue Usage
- Primary buttons
- Active states
- Progress indicators
- Links

### Forbidden
- Multiple accent colors
- Gradients
- Neon tones

---

## 4. Typography

### Font
- Inter ONLY

### Scale
- Display: 900
- Heading: 700
- Subheading: 500
- Body: 400

### Rules
- No mixing fonts
- Large text: negative tracking
- Small labels: positive tracking

---

## 5. Spacing System

### Base Unit
- 8px

### Scale
- 8 / 16 / 24 / 32 / 40 / 48

### Rules
- No arbitrary spacing
- Maintain vertical rhythm

---

## 6. Layout

### Page Structure
- Generous margins
- Max readable width
- Vertical stacking

### Cards
- Background: #F5F5F5
- Radius: 12px
- No heavy shadows

---

## 7. Components

### Buttons

Primary:
- Blue background
- White text
- Pill radius

Secondary:
- Outline or ghost

---

### Inputs
- Light surface background
- Clear borders
- Large tap targets

---

### Progress Bars
- Thin
- Blue fill only

---

## 8. Icons

- Line icons only
- Consistent stroke width
- No filled icons
- Blue only when active

---

## 9. Navigation

### Tab Bar
- Minimal
- Icon-first
- Label appears on active

---

## 10. States

### Loading
- Simple indicators
- No complex animations

### Empty
- Calm message
- Clear action

### Error
- Short message
- Immediate solution

---

## 11. Dark Mode

- Secondary priority
- Same system, darker base
- No new colors

---

## 12. What NOT to Do

- No gradients
- No multiple accent colors
- No heavy shadows
- No clutter
- No inconsistent spacing
- No random fonts

---

## 13. Final Principle

If a design decision makes the user think harder, it is wrong.

If it makes the experience clearer, it is correct.
